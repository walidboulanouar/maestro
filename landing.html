Maestro

The open-source version of Sakana's Fugu. An LLM routing brain: point your OpenAI or Anthropic client at it, and every request goes to the right model — cheap-first, verified, escalated only when needed — with the full route and cost shown. Self-hostable. MIT.

⭐ Star on GitHub Try in 30s

The problem

The best model for a one-line translation is not the best model for a distributed-systems design — but you pay frontier prices for both. Maestro sends the easy stuff to cheap models and only the hard stuff to the expensive ones, automatically.

How it works

task · difficulty
→
ROUTE
cheap→strong ladder
→
EXECUTE
via any gateway

accept / revise
↺
ANSWER
+ route + cost

A cheap model tries first. A verifier checks the answer. If it's not good enough, Maestro escalates to a stronger model (up to a turn limit). Built from the two Fugu papers — TRINITY + Conductor.

Try it in 30 seconds

Runs with zero API keys (built-in mock provider):

npx maestro serve
curl -s localhost:8080/v1/chat/completions -H 'content-type: application/json' \
  -d '{"model":"maestro-auto","messages":[{"role":"user","content":"hello"}]}' | jq .maestro

Add a key for real models — your OpenAI code doesn't change, just the base URL:

export OPENROUTER_API_KEY=sk-or-...     # open + closed, BYOK
# or LOCAL_OPENAI_BASE_URL=http://localhost:11434/v1   # Ollama/vLLM — 100% offline
